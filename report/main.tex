\section{Introduction}
Human activity recognition in videos is a core computer vision task with applications in surveillance, human-computer interaction, healthcare monitoring, and sports analytics. For this project, the objective is to classify six action categories from video using temporal template methods discussed in class.

Compared with deep video architectures, MHI-based approaches are lightweight, interpretable, and suitable for small/medium datasets. The central idea is to summarize recent motion into a single image where brighter pixels represent more recent movement. Shape descriptors computed on this template can then be fed to a classifier.

The contributions of this implementation are:
\begin{itemize}
    \item A fully constraint-compliant MHI pipeline (no banned Hu API, no deep learning framework).
    \item A robust feature extractor that improves invariance to noise, location, and scale.
    \item A unified multi-model evaluation framework with automatic best-model selection.
\end{itemize}

\section{Related Work}
Bobick and Davis introduced temporal templates (MEI/MHI) for representing human movement and recognizing actions with compact motion summaries \cite{bobick2001}. Moment invariants from Hu provide translation/scale/rotation-sensitive shape descriptors widely used for classical recognition tasks \cite{hu1962}. Modern action recognition methods rely heavily on deep architectures (e.g., two-stream networks, 3D ConvNets, transformers), but these are outside this assignment's allowed model family \cite{simonyan2014, carreira2017}. Therefore, this work focuses on classical temporal-template methods with careful feature engineering and model selection.

\section{Method}
\subsection{Problem setup}
Given an input video clip $V = \{I_t\}_{t=1}^{T}$, classify the observed human action into one of:
\[
\mathcal{Y}=\{\text{walking},\text{jogging},\text{running},\text{boxing},\text{waving},\text{clapping}\}.
\]

\subsection{Preprocessing and binary motion signal}
Each frame is resized to a fixed resolution (default $160\times120$), converted to grayscale, and lightly smoothed with a Gaussian filter to reduce compression/sensor noise.

For consecutive frames $I_t$ and $I_{t-1}$, the absolute difference is:
\[
D_t(x,y)=|I_t(x,y)-I_{t-1}(x,y)|.
\]
A binary motion image $B_t$ is then computed as:
\[
B_t(x,y)=
\begin{cases}
1,& D_t(x,y)\ge \theta_t,\\
0,& \text{otherwise},
\end{cases}
\]
where
\[
\theta_t = \max(\theta_{\min},\,\mu_D + 1.5\sigma_D),
\]
with $\mu_D,\sigma_D$ being the mean and standard deviation of $D_t$. This adaptive threshold floor is more robust than a fixed threshold under illumination and codec variation.

A morphological open-then-close operation removes small isolated noise and fills small holes. Finally, only the largest connected motion component is retained to suppress distractors.

\subsection{Motion History Image (MHI)}
The MHI update rule follows:
\[
M_t(x,y)=
\begin{cases}
\tau, & B_t(x,y)=1,\\
\max(M_{t-1}(x,y)-1,0), & B_t(x,y)=0.
\end{cases}
\]
Here, $\tau$ controls temporal memory length: larger $\tau$ preserves longer motion trails. After processing all frames, $M_t$ is normalized to $[0,1]$.

\subsection{MEI construction}
A Motion Energy Image (MEI) is also formed from the final MHI:
\[
E(x,y)=\mathbb{1}[M(x,y)>\delta],
\]
with default $\delta=0.05$. This complements MHI by encoding motion occupancy (binary) while MHI encodes recency/intensity (continuous).

\subsection{Motion-region canonicalization}
To improve translation and scale robustness, the motion region bounding box is extracted from MEI, padded, and resized to a canonical size ($96\times96$). This canonicalization is applied to both MHI and MEI before moment computation.

Two geometric summary features are also retained:
\begin{itemize}
    \item Bounding-box aspect ratio.
    \item Bounding-box area ratio relative to full frame.
\end{itemize}

\subsection{Moment features and manual Hu invariants}
Moments are computed directly (no banned helper API). Let raw moments be:
\[
m_{pq}=\sum_x\sum_y x^p y^q I(x,y),
\]
centroid
\[
\bar{x}=\frac{m_{10}}{m_{00}},\qquad \bar{y}=\frac{m_{01}}{m_{00}},
\]
central moments
\[
\mu_{pq}=\sum_x\sum_y(x-\bar{x})^p(y-\bar{y})^q I(x,y),
\]
and normalized central moments
\[
\eta_{pq}=\frac{\mu_{pq}}{\mu_{00}^{1+(p+q)/2}}.
\]

For orders $(p,q)\in\{(2,0),(1,1),(0,2),(3,0),(2,1),(1,2),(0,3),(2,2)\}$, both central and normalized moments are included. In addition, seven Hu invariants are manually derived from normalized moments. A signed-log transform
\[
f(v)=\operatorname{sign}(v)\log(1+|v|)
\]
is applied to stabilize magnitude ranges.

\subsection{Final feature vector}
The final descriptor concatenates:
\begin{itemize}
    \item Moment block from canonicalized MHI.
    \item Moment block from canonicalized MEI.
    \item Global motion statistics: motion ratio, mean MHI energy, standard deviation of energy, temporal density, aspect ratio, area ratio.
\end{itemize}
In the current implementation the feature dimension is 52.

\subsection{Classifier training and model selection}
Data split uses stratified 70/30 train/test partition. Four classifiers are trained and compared:
\begin{itemize}
    \item KNN ($k=3$) with standardization
    \item RBF-SVM ($C=3.0$) with standardization
    \item Random Forest (300 trees)
    \item Logistic Regression (max iterations 2000) with standardization
\end{itemize}
The best model is selected by lexicographic ranking: higher accuracy first, then higher macro-F1. The selected model is exported for inference.

\section{Error Analysis}
\subsection{Positive cases}
Typical successful cases include videos with stable backgrounds, clear single-subject motion, and moderate viewpoint changes. Under these conditions, temporal motion templates are highly discriminative.

\subsection{Negative cases}
Common failure modes on real data:
\begin{itemize}
    \item Camera motion introduces widespread false motion and corrupts $B_t$.
    \item Similar upper-body patterns (e.g., boxing vs. waving) overlap in moment space.
    \item Very short clips or low frame rates under-represent temporal trajectories.
    \item Extreme scale/viewpoint shifts reduce template consistency.
\end{itemize}

\subsection{Why robustness features help}
Compared with a basic MHI+moments baseline, the current pipeline improves stability through adaptive thresholding, largest-component filtering, canonicalized motion crops, and MHI/MEI feature fusion. These additions reduce background-noise sensitivity and improve cross-subject consistency.
